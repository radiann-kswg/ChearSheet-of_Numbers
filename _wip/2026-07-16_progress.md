### 2-2. Wikipedia 引用内の等式検証

ページ中の KaTeX 等式（`$n = 式$` 形式）を機械抽出し評価:

- 抽出できた **624 件中 622 件が真**（Wolfram でのスポット照合も全て True: 496 = Σ(1..31) = U₃(4) = 5⁴−5³−5+1 = 16·σ(16)、57 = 7⁰+7¹+7²、57² = Σσ(k)−Σk (k=1..100) など）
- **偽の等式 2 件**（いずれも整形パイプラインが生成した誤り。出典側は正しい）:
  - `numbers/3xx/388.md`: 「$388 = 4^{2} \times 97$」→ 正しくは $2^{2} \times 97$
  - `numbers/7xx/715.md`: 「$715 = 714$」→ 原文「714と715はルース＝アーロン・ペア」の式と文の境界崩れ

### 2-3. 発見した品質課題（優先度順）

1. **KaTeX 整形の破損**: HTML 実体参照（`&times;` `&minus;`）が未デコードのまま `$...$` 内に残存 — **196 ページ**（うち数式として破綻 185 ページ）。式と後続説明文の連結（「$57 = 2^{6} &minus; 2^{3} + 1 n = 2$ のときの…」等)も同根。
   - 対処: `tools/wikipedia_ja.py` で (a) 実体参照のデコードを KaTeX 変換前に実施、(b) 数式ノードと平文の境界を保持。
2. **HTTP ステータスコード節のリンク過剰生成**: 100〜599 の**全 500 ページ**に MDN リンクを出力しているが、実在コードは約 63。**約 437 件がリンク切れ**（例: `/Status/496` は存在しない）。
   - 対処: 実在コード一覧（IANA / 参照先の CheatSheet-of_HttpResponceDataCode）でゲートし、非該当は「該当なし」表記へ。
3. 上記 2-2 の偽等式 2 件（1. の修正で同時に解消される見込み）。

いずれも修正は生成スクリプト側で行い全数再生成が必要（変更量が 500 行を大きく超えるため、リポジトリ規約に従い実行前にユーザー確認を要する）。

## 4. 追記（同日後続作業）: 修正対応と記述強化の実施結果

### 4-1. 引用整形パイプラインの修正（tools/wikipedia_ja.py / tools/generate_numbers.py）

- HTML 実体参照（`&times;` `&minus;` 等）を `html.unescape`（二重適用）でデコード。抽出側（将来のキャッシュ更新用）と生成側（既存キャッシュの浄化 `_sanitize_excerpt`）の両方に実装。
- `_split_math_prefix` を改善:
  - 式の完了後に始まる「n = 2 のときの…」等の説明文を式から分離
  - 上付き指数の平坦化破損（「2 2 + 4 2」← 2²+4²）を修復
  - 演算子を挟まない数値項の連続で説明文断片を切断
  - 波括弧/丸括弧が不均衡な断片は整形しない（壊れた KaTeX の防止）
- **算術検算ゲート**を新設: 引用先頭の純算術等式を評価し、偽なら掲載しない（Wikipedia 原文の誤植 388=4²×97 も検出・排除）。巨大な冪による停止を防ぐ**有界 AST 評価器**（指数 ≤64、値 ≤10^18）を使用。破損テキスト由来の `3^5 0^5`→`3**50**5`（3 の約 3 億乗）でハングする問題も解消。
- 既知の破損定型（「715 = 714と715は…」「57 = = 素数…」）の修復規則を追加。

### 4-2. HTTP ステータス節の実在コードゲート（tools/generate_numbers.py）

- `HTTP_STATUS_PAGES`（実在コード 87 件: IANA 標準＋nginx/Cloudflare/プロキシ慣行の非標準）を新設。
- 実在コード → MDN（個別ページがあるもののみ）＋ CheatSheet-of_HttpResponceDataCode の個別ページへ**ディープリンク**。非標準コードはその旨を明記。
- 非該当の 100〜599 → 「割り当てなし」の説明＋IANA レジストリへのリンク（RFC 9110 の x00 等価扱いに言及）。

### 4-3. 記述強化（Wolfram コネクタ活用）

- **数学的性質の拡充**: 「その他の性質」行を新設（回文数/回文素数/エマープ/双子素数/ソフィー・ジェルマン素数/安全素数/半素数/楔数/ハーシャッド数/ハッピー数/高度合成数/階乗数/リュカ数/ペル数/カタラン数）。全集合を Wolfram Language で検算し完全一致（ハッピー数142・ハーシャッド数212・半素数299・楔数135・双子素数69・SG素数37・安全素数25・エマープ36・回文数99 ほか）。

  - 小惑星番号 1〜999 の名称（例: 57 Mnemosyne、496 Gryphia）＋ JPL Small-Body Database への一次情報リンク
  - NGC 1〜999 の天体分類（銀河/星団/星雲/恒星等）＋ NGC 天体一覧へのリンク
