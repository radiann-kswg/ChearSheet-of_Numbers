## 1. フラグの拡張と解説の集約（全 1,000 ページ再生成）

### 概要セクションの新形式

フラグを 4 分類で列挙し、それぞれの解説先セクションを明記:

- **フラグ（数学）**: 平方数・三角数・完全/過剰/不足数・半素数・楔数・ハーシャッド数・ハッピー数・回文数・エマープ・双子素数・SG/安全素数・高度合成数・階乗数・リュカ/ペル/カタラン数 など → 解説は『数学的性質』
- **フラグ（科学・技術）**: 原子番号（元素名つき）・小惑星・NGC天体（分類つき） → 解説は『科学・技術』
- **フラグ（規格・コード）**: HTTPステータスコード（実在コードのみ）・ISO 3166-1・国番号（E.164）・7400シリーズ → 解説は『規格・コード』
- **フラグ（文化・いわれ）**: 数秘術・エンジェルナンバー・ヘブライ数字・ゲマトリア（著名な数価）・吉凶・忌み数・伝承・神話・名数・番号のいわれ・語呂合わせ・スラング・創作作品 → 解説は『数秘・占術・文化のいわれ』

### 『数学的性質』に「フラグの解説」を新設

数学フラグ全種に **定義＋その数での根拠** を機械導出で付与（例）:

- 不足数: 約数の総和が自身の 2 倍に満たない数。σ(57) = 80 < 114 = 2 × 57
- 半素数: ちょうど 2 つの素数の積で表される合成数。57 = 3 × 19
- 三角数: 1 から連続する自然数の和で表される数。496 = 1 + 2 + ⋯ + 31（第 31 三角数）
- ハッピー数: 定義＋ 1 に到達する系列（短い場合は全系列を表示）
- 従来の「その他の性質」1 行はここに統合・置換

### 解説の重複解消

- Wikipedia『性質（要約）』冒頭の機械由来の再掲行（「合成数（Wikipediaの『性質』参照）」「不足数（Wikipedia参照）」「平方数」等）を全廃し、Wikipedia 固有の情報（「n 番目の素数として説明される」・短い引用）のみを残した
- 機械行のみだった 363 ページには「『数学的性質』のフラグの解説に集約」の案内行を表示
- 検証: 全 1,000 ページで「フラグ（数学）に列挙された名前 ⇔ フラグの解説の項目」の対応一致（不一致 0）

## 2. GitHub Pages ビルドエラーの解消

### 原因（特定済み）

`numbers/0xx/023.md`（誕生日のパラドックスの式 `\frac{{}_{365}...`）と `numbers/1xx/105.md`（`\frac {105}{{\pi}^{4}}`）に含まれる **`{{`** を、Jekyll の Liquid が変数開始と誤認し「Variable was not properly terminated」で build ジョブが失敗（Jekyll の既知問題: jekyll/jekyll#5458, #9487）。

### 対処（Jekyll 廃止 → 静的配信へ）

リポジトリには `index.html`＋`assets/`（Numbers Lore Viewer。**生の .md を fetch して描画するクライアントサイドビューアー**）が存在するため、Jekyll でビルドすると (a) `.md` が `.html` に変換されてビューアーの fetch が 404 になる、(b) `index.md` と `index.html` が衝突する、という根本的な不整合がある。よって Liquid エラーの回避に留まらず、**Pages を Jekyll なしの静的配信に切り替えた**。

- `.github/workflows/jekyll-gh-pages.yml` を書き換え: `jekyll-build-pages` を廃止し、rsync による静的コピー（`tools/`・`_wip/`・`_template/`・`.private/`・`AGENTS.md`・`CLAUDE.md` 等は除外）＋`.nojekyll` で `upload-pages-artifact` する構成へ。`actions/checkout@v4 → v5`（Node.js 20 非推奨警告の解消）
- ビューアーの調整:
  - `index.html`: KaTeX 0.16.11（css/js/auto-render）を追加読込
  - `assets/app.js`: 描画後に `renderMathIn()` で `$...$` / `$$...$$` を KaTeX 描画（DOMPurify サニタイズ後に適用）。`index.md` への内部リンクもビューアー内で遷移できるよう `normalizeDocPath` を拡張
  - `tools/build_viewer_index.py` 新設: `assets/numbers-index.json`（検索インデックス）を最新のページ内容から再生成。`generate_numbers.py` 実行時に自動同期。性質フィルタを 13 → 28 種に拡張（半素数・楔数・ハーシャッド数・ハッピー数・回文数・エマープ・双子素数・SG/安全素数・高度合成数・階乗数・リュカ/ペル/カタラン数 を追加）
- `generate_numbers.py` の出力段で `{{` → `{ {`、`{%` → `{ %` に無害化（全 1,000 ページで Liquid 危険列 0。静的配信では必須ではないが、将来 Jekyll 化する場合への備え）
- 一旦作成した Jekyll 用の `_config.yml`・`_includes/head-custom.html` は不要になったため削除（`_to_delete/` へ退避）
- `README.md` に GitHub Pages の閲覧 URL を追記

### ビューアーの動作検証（ヘッドレス Chromium・静的サーバー）

- 57 ページ表示: KaTeX 数式 7 箇所を描画、`半素数` フィルタで 299 件（2026-02 の Wolfram 検算値 299 と一致）、内部リンク 056 への遷移、index.md ルーティング — すべて成功、コンソールエラー 0

## 3. 追記（同日後続作業）: ビューアーの数式乱れ修正とサイドバー改修

### 数式の乱れの原因と対処

- **原因1（ほぼ全ページ・996 ページ）**: `$\sigma(57) = 80 < 114$` のような数式内の `<` を marked が HTML タグ開始と誤認し、DOMPurify のサニタイズで後続が欠落していた。`_` の対（強調記法）と衝突する数式も 20 件。
  - 対処（ビューアー）: `assets/app.js` に**数式保護**を実装。Markdown 変換前に `$...$` / `$$...$$` を私用領域文字のトークンへ退避し、サニタイズ後にテキストノードへ復元してから KaTeX で描画（`extractMathSpans` / `restoreMathSpans`）。
  - 対処（生成側・多重防御）: フラグの解説の比較演算子を `\lt` / `\gt` に変更。
- **原因2（1 件）**: `numbers/1xx/105.md` の引用が抽出時から壊れており（`\left` と `\right` の不均衡）、疑似数式として表示されていた。
  - 対処: `_katex_balanced()` ゲートを新設し、`\left`/`\right`・波括弧が不均衡な式は整形せず素の引用のまま掲載（全数再生成で不均衡スパン 0 を確認）。

### サイドバー改修（ユーザー選択: 別スクロール＋番号ジャンプ）

- フィルタ枠（`.filter-scroll`、最大 34vh）と数字の索引（`.doc-scroll`）を**それぞれ独立のスクロール領域**に分離（サイドバー全体は flex 縦積み・overflow hidden）
- サイドバー上部に**番号ジャンプ欄**（0〜999 の直接入力、Enter / 「開く」ボタンで該当ページへ）を追加

### 検証（ヘッドレス Chromium）

- σ 行の完全表示・KaTeX 描画 / フィルタ・索引の独立スクロール / 496 への番号ジャンプ — すべて成功、コンソールエラー 0
